PyTest is a testing framework that allows users to write test codes using Python programming language. It helps you to write simple and scalable test cases for databases, APIs, or UI. PyTest is mainly used for writing tests for APIs. It helps to write tests from simple unit tests to complex functional tests.

In my project I created 5 webpages with one homepage and 4 pages explaining the stuff I learnt in the web systems development class. I used Z layout to create the home page and F layout for the remaining pages. To test the webpages used a python file called pages_test.py in this file I defined several functions to test each webpage response and data of the webpages.

In the project file <- we have .github file in which we have dev.yml and prod.yml.

dev.yml <- will have developing code in which we will link our Heroku and Docker container where test runs on ubuntu

Prod.yml file <- will be running on ubuntu it has all the access to Heroku and docker hub whatever is tested and satisfied with dev app will be finally deployed in the prod app.

In app folder <- we have our pages, inside pages we have all the 5 html files of the webpages

In tests folder <- we have init.py which will run the tests and testing.py which will test the webpages we created in the app templates.

docker-compose.yml <- Orchestration file that contains the configuration to develop locally, it overrides the Dockerfile to run the flask development server instead of running the gunicorn server that is used for hosting on Heroku

Dockerfile <-File used to create an image to run in a container that runs the program

Readme file <- this is used to see the work in the summarized manner and some important things like how to start the app and how to use the app will be mentioned inside this
